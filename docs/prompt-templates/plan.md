## 约束

1. **只规划不实现**：输出设计方案和步骤，不要写实现代码
2. **参考架构文档**：方案必须与 ARCHITECTURE.md 中的设计一致，如需偏离必须说明原因
3. **小步验证**：方案中应包含验证点（Checkpoint），尽早验证关键假设
4. **接口先行**：先定义模块间的接口契约，再规划内部实现
5. **任务粒度**：每个任务点应是一个可独立验证的原子变更，能在一次 commit 中完成

## 输出格式

按以下格式输出规划结果：

1. **方案概述**：一句话描述整体思路
2. **任务列表**：按执行顺序排列，每个任务包含：
   - 任务描述
   - 涉及的文件
   - 验证标准
   - 与其他任务的依赖关系
3. **接口契约**：模块间需要约定的接口定义
4. **风险点**：可能遇到的问题和应对方案
