## 通用约束

1. **先读后改**：修改文件前先 Read 确认当前内容，不做盲改
2. **文件小巧**：单文件不超过 300 行，超过时考虑拆分
3. **遵循已有模式**：命名、结构、错误处理方式与现有代码一致
4. **不做过度设计**：不写"以后可能会用到"的代码，不引入任务范围外的重构
5. **安全第一**：不引入命令注入、SQL 注入、XSS 等安全漏洞
6. **函数有描述**：每个函数需要 docstring 描述其用途，复杂逻辑需要注释说明意图
7. **日志与异常**：关键处理环节记录日志，异常捕获信息要包含足够的上下文（堆栈、参数、状态）
8. **清理 import**：写完后确认所有 import 都有使用，删除未使用的 import
9. **编写测试**：功能代码完成后，为关键路径编写冒烟测试，覆盖正常流程和边界情况
10. **运行测试**：写完后运行测试确认通过，不要报告"应该能通过"
