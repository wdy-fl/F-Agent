## 审查清单

### 功能正确性
- [ ] 实现是否满足设计文档中的要求
- [ ] 是否处理了关键的边界情况
- [ ] 工具调用的参数是否做了合理校验

### 代码质量
- [ ] 单文件不超过 300 行
- [ ] 函数有 docstring 描述用途
- [ ] 无未使用的 import
- [ ] 日志记录覆盖关键处理环节
- [ ] 异常捕获包含足够的上下文信息

### 安全性
- [ ] 无命令注入风险（terminal 工具的 shell=True 需特别关注）
- [ ] 无 SQL 注入风险
- [ ] 无路径遍历风险
- [ ] 文件操作已做路径校验

### 测试覆盖
- [ ] 关键路径有冒烟测试
- [ ] 边界情况有测试
- [ ] 所有测试通过

### 与已有代码的一致性
- [ ] 命名风格与现有代码一致
- [ ] 错误处理方式与现有代码一致
- [ ] 返回值格式与已有模块兼容

## 输出格式

按以下格式输出审查结果：

1. **通过项**：列出满足标准的点
2. **问题项**：列出未满足标准的点，每个问题包含文件路径、行号、问题描述、建议修复方式
3. **总结**：整体评价和是否建议通过
