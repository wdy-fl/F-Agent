# 阿福提示词记忆能力描述超前问题记录

## 背景

当前阿福处于 Phase 1 MVP 阶段，已经实现最小版会话持久化：对话消息可以写入 SQLite 的 `sessions` / `messages` 表。

但 Phase 2 记忆与用户建模尚未实现，包括：

- FTS5 全文搜索召回；
- `MemoryManager`；
- `<memory-context>` 上下文围栏；
- USER.md 用户画像；
- `tools/memory.py` 记忆工具；
- 长对话上下文压缩。

同时，当前系统提示词和阿福回复中会声称自己具备“跨对话记忆”能力，导致用户预期与真实能力不一致。

## 现象

用户询问阿福是谁、是否知道用户是谁时，阿福会出现类似表述：

```text
我可以帮你做很多事情：
- 记住偏好：跨对话记住你的习惯和需求
```

或：

```text
我的记忆能力可以跨对话保留，这样下次见面我就能认出你了！
```

但当前真实代码尚不能跨会话召回历史，也不能自动维护 USER.md 用户画像。

## 初步根因

当前 `agent/prompt.py` 的 `AGENT_IDENTITY` 中包含能力描述：

```text
你是阿福（F-Agent），一个具备对话记忆和技能自创能力的个人智能助手。

## 核心能力
- 持久记忆：跨会话记住用户偏好和历史对话
- 技能自创：完成复杂任务后提炼可复用技能
```

这属于目标架构描述，但已经注入到当前运行时 system prompt 中。模型会基于 system prompt 对外承诺这些能力，即使对应实现尚未完成。

当前真实能力边界：

- 已具备：SQLite 会话和消息落库。
- 已计划：Phase 2 实现跨会话记忆和用户建模。
- 未具备：自动搜索历史、注入记忆上下文、更新用户画像、通过 memory 工具保存偏好。

因此，问题不是模型“胡说”，而是 system prompt 暴露了尚未实现的路线图能力。

## 涉及文件

- `agent/prompt.py`
  - `AGENT_IDENTITY` 中的核心能力描述。
  - `MEMORY_GUIDANCE` 中的 memory 工具使用规则。
  - `build_system_prompt()` 的 `include_memory_guidance` 参数。

- `docs/ARCHITECTURE.md`
  - 描述目标架构中的完整记忆系统。

- `docs/superpowers/plans/2026-05-15-phase2-memory-plan.md`
  - 描述后续 Phase 2 记忆与用户建模实现计划。

## 建议修复范围

该问题建议作为“提示词与当前实现能力对齐”任务单独处理。

可选方案：

### 方案 A：Phase 1 阶段降级运行时能力描述

将运行时 system prompt 改为只描述当前已实现能力，例如：

```text
你是阿福（F-Agent），一个本地 CLI 智能助手。
当前你支持智能对话、工具调用和会话记录保存。
跨会话记忆、用户画像和技能自创仍在后续阶段实现中，不能声称已经具备。
```

优点：最符合当前真实能力，避免误导用户。

缺点：需要在 Phase 2 完成后再升级提示词。

### 方案 B：区分“目标能力”和“当前能力”

在 system prompt 中明确写出：

```text
目标能力包括持久记忆和技能自创，但当前版本尚未完成这些能力。回答用户时只能基于当前已实现能力说明。
```

优点：保留产品愿景，同时降低误导。

缺点：提示词更长，模型仍可能偶尔提及目标能力。

### 方案 C：等 Phase 2 实现后再处理

不修改当前 prompt，等完整记忆系统实现后自然对齐。

优点：短期无需修改。

缺点：在 Phase 2 完成前会持续误导用户，不推荐。

## 推荐方向

推荐采用方案 A：Phase 1 阶段降级运行时能力描述。

原则：运行时 system prompt 只描述“当前真实可用能力”，目标能力留在架构文档和路线图文档中。

## 测试建议

- 为 `build_system_prompt()` 增加断言：默认 prompt 不应声称“跨会话记住用户偏好和历史对话”。
- 如果保留 memory guidance 参数，则测试默认情况下不注入 `MEMORY_GUIDANCE`。
- 如未来 Phase 2 完成，再调整测试和 prompt。

## 非目标

本问题不处理：

- 实现 Phase 2 记忆系统。
- 实现 USER.md 用户画像。
- 实现 memory 工具。
- DeepSeek thinking mode 的 `reasoning_content` 丢失问题。
- CLI 输出重复问题。

## 当前状态

- 问题已在真实 CLI 回复中复现。
- 当前 system prompt 描述与 Phase 1 实现能力不一致。
- 尚未修复。
- 应作为独立任务处理，优先级建议高于 Phase 2 大规模实现，因为它会直接影响用户预期。
