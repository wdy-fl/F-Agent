# Phase 2 功能验证清单

> 逐一验证 Phase 2 新增的 6 个功能模块，每项通过后打勾。

---

## 1. FTS5 全文索引 + 搜索召回

- [ ] **1.1 DB Schema 升级** — 删除 `workspace/state.db`，启动 `python3 main.py` 输入任意消息后退出，确认 `state.db` 被重新创建且未报错
- [ ] **1.2 搜索历史消息** — 与阿福进行 3-5 轮包含特定关键词的对话（如"Python 异步编程"），然后新开会话让阿福用 memory 工具搜索该关键词，确认能召回
- [ ] **1.3 v1 数据库迁移** — 如果有旧的 Phase 1 数据库，启动后确认 `schema_version` 变为 2，FTS5 表自动创建

## 2. 上下文围栏

- [ ] **2.1 围栏注入** — 完成几轮对话后开启新会话，观察阿福能否引用上一会话的内容（说明记忆被 `<memory-context>` 标签封装后注入）
- [ ] **2.2 围栏剥离** — 阿福回复中不应出现 `<memory-context>` 标签原文（围栏仅内部使用，用户不可见）

## 3. 记忆管理器

- [ ] **3.1 预取触发** — 新会话开始后，阿福应自动搜索与当前话题相关的历史对话并个性化回复
- [ ] **3.2 历史话题关联** — 对阿福说"还记得我之前说的吗？"，看是否能召回相关历史内容

## 4. 记忆集成到主循环（集成测试）

- [ ] **4.1 记忆注入到 LLM 可见** — 观察阿福在多轮对话中是否能引用本会话的早期内容（说明系统提示词中包含记忆指引）
- [ ] **4.2 DB 存储原始消息** — 用 SQLite 查看 messages 表，确认 user 消息不含 `<memory-context>` 标签（保证搜索干净）

验证命令：
```bash
sqlite3 workspace/state.db "SELECT content FROM messages WHERE role='user' LIMIT 5;"
```

## 5. 用户建模

- [ ] **5.1 USER.md 创建** — 告诉阿福几个个人偏好（如"我喜欢用 VS Code 写 Python"），然后让其调用 memory 工具保存画像，确认 `workspace/USER.md` 文件被创建
- [ ] **5.2 LLM 驱动画像更新** — 再次告诉阿福新的偏好（如"我现在改用 PyCharm 了"），让其更新画像，确认 USER.md 被合并更新而非覆盖
- [ ] **5.3 画像注入** — 开启全新会话，阿福应能在回复中体现你的偏好

验证命令：
```bash
cat workspace/USER.md
```

## 6. 上下文压缩

- [ ] **6.1 压缩触发** — 进行约 30+ 轮的密集对话（包含大量工具调用结果），观察日志中是否出现 "Compressed" 信息
- [ ] **6.2 压缩后对话连贯** — 压缩触发后，阿福应仍能正确理解会话上下文，回复不受压缩影响

验证命令：
```bash
tail -f workspace/logs/agent.log | grep -i compress
```
